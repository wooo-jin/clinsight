# Clinsight

**Claude Code 세션을 자동으로 기록하고, 분석하고, 복리화하는 TUI 대시보드**

---

Claude Code를 쓰다 보면 이런 경험, 한 번쯤 있을 겁니다:

- 어제 Claude랑 뭘 논의했는지 기억이 안 남
- 세션이 날아가서 처음부터 다시 설명
- "그 프롬프트 또 써먹을 수 있는데..." 하고 찾지 못함
- 한 세션에 $30 넘게 쓴 걸 뒤늦게 발견

Clinsight는 이 문제를 해결합니다.

## 뭘 해주나요?

### 1. 자동 세션 아카이브

Claude Code의 Hook 시스템을 활용해 **모든 대화를 자동으로 기록**합니다. 세션이 시작되면 기록을 시작하고, 끝나면 완전한 아카이브를 생성합니다. 별도 조작 없이 그냥 Claude Code를 쓰면 됩니다.

### 2. TUI 대시보드

터미널에서 바로 확인하는 6개 탭 대시보드:

```
 Clinsight  | 42 sessions | updated 17:30

 📊 Dashboard  💡 Insights  📋 Sessions  💰 Cost  📦 Compound  📂 Archive
```

- **Dashboard** — 오늘/주간/월간 요약, 비용 추이 그래프, 효율 점수
- **Insights** — 삽질 탐지, 컨텍스트 포화도 경고, 개선 제안
- **Sessions** — 세션별 상세 분석 (토큰, 도구 사용, 편집 파일)
- **Cost** — 모델별/프로젝트별 비용 분석
- **Compound** — AI 기반 패턴 추출 및 복리화
- **Archive** — 저장된 세션 대화 전문 조회

### 3. 세션 분석 엔진

각 세션을 자동으로 분석해서 **효율 점수**를 매깁니다:

- **삽질 지수** — 편집을 되돌린 횟수로 실제 삽질을 탐지 (단순 반복 편집 ≠ 삽질)
- **탐색 효율** — 읽기만 하고 편집에 기여하지 않은 파일 비율
- **컨텍스트 포화도** — 200K 윈도우 대비 실제 사용량, compact 타이밍 제안
- **1회 해결률** — 되돌림 없이 완료된 편집 비율
- **비용 경고** — 세션당 $10/$30 초과 시 Claude에게 자동 알림

### 4. Compound (복리화)

매일 밤 크론잡이 돌면서 그날의 세션을 Claude에게 보내 심층 분석합니다:

- **패턴** — 반복되는 작업 방식 인식
- **솔루션** — 재사용 가능한 해결책 축적
- **컨벤션** — 코딩 관례 자동 추출
- **골든 프롬프트** — 효과적이었던 프롬프트 보존
- **예방 규칙** — 실수에서 배운 규칙 생성 (CLAUDE.md에 추가 가능)

> 오늘의 삽질이 내일의 자산이 됩니다.

### 5. 실시간 알림 (Claude 컨텍스트 주입)

매 프롬프트마다 현재 세션을 분석해서, 문제가 감지되면 Claude의 컨텍스트에 경고를 자동 주입합니다. Claude가 이를 인식하고 응답에 자연스럽게 반영합니다.

- **컨텍스트 75%+** → "곧 /compact가 필요할 수 있습니다"
- **컨텍스트 90%+** → "/compact 실행을 권장합니다"
- **비용 $10+** → "비용이 높아지고 있습니다"
- **되돌림 3회+** → "접근 방식을 재검토하는 것이 좋겠습니다"
- **60분+ 세션** → "40-50분 단위 세션이 효율적입니다"

별도의 알림창이 뜨는 게 아니라, Claude가 알아서 언급해줍니다.

### 6. 좀비 세션 탐지

고아 프로세스나 방치된 세션 디렉토리를 찾아내고 정리합니다.

## 기술 스택

- **Ink + React** — 터미널 UI 프레임워크
- **TypeScript strict** — 전체 타입 안전성
- **FSD 아키텍처** — Feature-Sliced Design 패턴
- **Claude Code Hooks** — 세션 라이프사이클 연동
- **Vitest** — 83개 테스트 통과
